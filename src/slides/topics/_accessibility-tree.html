What Is the Accessibility Tree?

The accessibility tree is a subset of the DOM that assistive technologies can access and interpret.

How It Works

When the browser parses the DOM, it builds:

  • The visual rendering tree (what you see)
  • The accessibility tree (what screen readers interpret)

DOM vs Accessibility Tree

DOM Tree Accessibility Tree
All elements Only relevant elements
Styling and scripting Semantics and interaction
Full page content Filtered for accessibility roles

What Gets Included?

  • Semantic HTML elements (e.g., <button>, <nav>)
  • Elements with meaningful roles (role="dialog")
  • Content with accessible names or labels
  • Focusable and interactive elements

What Gets Excluded?

  • display: none or visibility: hidden
  • aria-hidden="true"
  • Non-semantic <div> or <span> without role or content
  • Elements without labels, roles, or focus

Example Comparison

<div>Hello</div>
<button>Submit</button>
<div aria-hidden="true">Hidden info</div>

In the accessibility tree, only the <button> appears.

Why It Matters

The accessibility tree is what screen readers "see".
If it's not there, it doesn't exist for users who rely on assistive technology.

How to Inspect It

  • Chrome DevTools: Accessibility panel
  • Firefox: Accessibility Inspector
  • axe DevTools browser extension
